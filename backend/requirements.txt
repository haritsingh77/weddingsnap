# ── Deliberately NOT here ────────────────────────────────────────────────────
#
#   face_recognition / dlib      compiled from source in the Dockerfile, and
#                                unused since the move to InsightFace
#   insightface, onnxruntime     ~1.8 GB resident once the model loads
#   scikit-learn                 only the DBSCAN fallback for clustering
#   opencv-python-headless       only used by the video frame sampler
#
# The server no longer recognises faces. Clustering happens once during
# preprocessing on the machine with the GPU, and the answer is stored in
# faces.cluster_id — so the API reads a table rather than re-deriving it, and
# guests identify themselves with a per-guest link rather than a selfie.
#
# This is what makes the image small enough for a free host, and it is why the
# API cannot enrol a new face. To add someone: preprocess locally, name their
# cluster, sync, and send them a link. Install requirements-preprocess.txt for
# that side of the work.
